## 검증된 핵심 (confidence high)

1. **리스트+필터+정렬+검색은 하나의 상호의존 시스템** — 가장 약한 고리가 전체를 좌우. 시각이 아니라 사용성 테스트로 검증해야 함(잘생긴 화면이 잘 작동하는 건 아님). ⚠️ "mediocre=67~90% vs 최적화=17~33% 이탈"은 **사용성 테스트 과제 이탈률**(매출 전환율 아님). [baymard/ecommerce-product-lists, /eCommerce-search]

2. **드롭다운 = Shell Approach.** 풀 커스텀 드롭다운은 **31%가 심각한 접근성 결함**(키보드 탭/포커스/타입어헤드/높이/방향 5종). → **닫힌 상태만 커스텀, 열면 네이티브 패널**(80% 효과/20% 비용, 선택 로직은 네이티브 HTML). **반드시 키보드 포커스 스타일**을 줄 것(대부분 구현이 빠뜨림). *→ "둥글게 했는데 클릭하면 네모"는 네이티브 패널이라 정상·접근성 OK. 고칠 건 닫힌 상태.* [baymard/custom-dropdowns] (⚠️ 31%/82% 수치는 체크아웃 벤치마크 기준 — 메커니즘은 동일 전이)

3. **검색 — 눈에 띄는 인라인 = 검색 사용↑, 돋보기 아이콘만 = 카테고리 탐색 유도.** 시각 중심 패션은 검색을 덜 강조하는 게 정당(Urban Outfitters는 일부러 돋보기만). 단 *의식적 트레이드오프*. [baymard/search-field, nng/magnifying-glass-icon]

4. **데스크톱 레이아웃 — 좌측 사이드바가 정설(~80%)**, 적용 필터·매치수 상시 노출. **단 패션 예외**: 필터 6~8개로 적으면 **상단 가로 툴바가 사이드바를 이길 수 있음**(가로공간→큰 썸네일, 시각 비교 우위). → 두 출처가 *패션에선 갈림*. [baymard/horizontal-filtering, /current-state-product-list]

5. **적용된 필터 = 제거 가능한 칩으로 상시 표시**(전체해제 + 항목별 해제, 큰 히트영역·명확한 ✕). 카운트만("Filter (3)")은 나쁨. 20~28% 사이트가 이 overview 누락. [baymard/how-to-design-applied-filters]

6. **적용 타이밍은 디바이스별** — 데스크톱=**즉시(real-time)**, 모바일=**"N개 결과 보기" 배치 버튼**(중간 리프레시 혼란 방지). 본질은 사용자 의도(탐색형=즉시, 다중조건형=배치). [baymard/filter-ui, nng/applying-filters]

7. **모바일 = 풀스크린/바텀시트 드로어**(사이드바 욱여넣기 X). 모바일 PLP UX가 업계 전반 더 나쁨(78% vs 58% poor/mediocre)라 별도 주의. [baymard, nng/mobile-faceted-search]
