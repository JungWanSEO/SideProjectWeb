# 개발 일지 작성법

`docs/dev-log.md`에 기록한다. 목적: **면접용 근거**(개발기간·결정·문제해결 스토리) + **Claude 그라운딩**(이전 결정 유지).

## 형식
날짜별 섹션 안에 아래 3개 카테고리를 둔다 (해당 없으면 생략):

```markdown
## YYYY-MM-DD (N일차)
### 추가
- 새로 만든 것 (기능 / 파일 / 설정)
### 결정
- 무엇을 선택했고 **왜** 선택했는지  ← ★ 면접 단골, 가장 중요
### 문제 / 해결
- [증상] → [원인] → [해결] → [배운 점]
```

## 좋은 엔트리 규칙
- **"무엇을"보다 "왜"를 적는다.** 모든 결정엔 이유를 붙인다.
- 문제는 반드시 `증상 → 원인 → 해결 → 교훈` 순서로. 면접에서 그대로 답변 소재가 된다.
- 날짜는 절대값(`YYYY-MM-DD`) + `N일차`를 병기해 총 기간 계산이 쉽게.
- 한 줄은 간결하게. 길어지면 `docs/architecture.md`로 빼고 여기선 링크.

## 작성 후
- **큰 결정**(스택/구조 변경 등)이면 `CLAUDE.md`와 `docs/architecture.md`에도 반영 검토.
- 날짜는 추측하지 말고 사용자에게 확인하거나 시스템 날짜를 따른다.
